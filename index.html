Step 1 — Define Organisation Space Master Catalog

Define the master list of VM modules (Space Master) available across the organisation. Each module has a category (Wall, Table, Bin, Platform, Fridge…), a size code, an assortment type (Core, Campaign, Fixed…), MDQ/PDQ capacity values, and validity dates.

Step 2 — Record Store Space (Modules × Locations × Count)

For each store, allocate physical locations to modules. Each row specifies which module occupies a space, where it sits (P1, P2, etc.), its physical dimensions, and critically the fixture count — how many units of that module exist at that location. The count directly drives how many items can be ranged into that space (count × MDQ).

Step 3 — Item ↔ Space Master Eligibility

Each item declares which Space Master module(s) it is eligible to be displayed in (FK → SpaceMaster.moduleID), along with its own item-level MDQ/PDQ. This mapping is the key input that determines which items can be ranged into which store spaces. An item's Space Master eligibility combined with a store's module allocation produces the candidate assortment.

Step 4 — Merge Store Space × Item Eligibility → Store Assortment

The merge engine joins Store Space (which modules exist at which locations with what count) with Item-Module eligibility (which items can go into which modules). The result is the Store Assortment — the full list of items rangeable into each store space. Capacity utilisation is calculated as items ranged vs. total MDQ capacity per space.
